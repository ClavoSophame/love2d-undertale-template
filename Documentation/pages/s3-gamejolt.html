Gamejolt API

您是否有想过——当玩家的数据不能存在本地（可能电脑被清空），而他们想要获取自己之前的进度？
亦或者是您想要展示/记录玩家的成就/得分排行榜？

Gamejolt提供了天然的服务器环境可供使用。

初始化Gamejolt绑定游戏

使用代码gamejolt.init(app_id, private_key)来绑定你自己的游戏

  • app_id - 游戏的id（六位数字）。
  • private_key - 游戏的token。
示例 
                
-- 绑定自己的游戏到gamejolt。
gamejolt.init(969905, "oooidktheyarejusttoolongformetoclone")

使用代码gamejolt.auth_user(username, user_token)进行用户绑定。

  • username - 用户的名称。
  • user_token - 用户的TOKEN（不是密码，要密码是违规的）。
示例 
                
-- 绑定用户账户信息。
gamejolt.auth_user("Clavo-Sophame", "AMINUOSASC")

上传/读取玩家的所有数据到Gamejolt

成就的获取与发放（Trophy/Trophies）

使用gamejolt.unlock_achievement(trophy_id)给予玩家成就。

  • trophy_id - 奖杯（成就）的ID。
                
-- 给予用户id为100000的成就。
gamejolt.unlock_achievement(100000)

使用gamejolt.fetch_achievements(achieved_only)获取玩家当前所获得的所有成就。

  • achieved_only - 是否只获取玩家获得的，而不是全部的。
                
-- 获取所有成就。
gamejolt.fetch_achievements()

-- 只获取玩家获得的所有成就。
gamejolt.fetch_achievements(true)

游戏状态的显示（Session）

Gamejolt要求游戏在运行时维护一个会话（Session），用于显示玩家当前的在线状态。
主要有以下几个函数：

  • gamejolt.session_open() - 打开一个新的游戏会话，通常在游戏启动时调用。
  • gamejolt.session_ping(status) - 向服务器发送心跳包，保持会话活跃。
    status参数可以为"active"（活跃）或"idle"（空闲）。
  • gamejolt.session_close() - 关闭当前会话，通常在游戏退出时调用。
示例 
            
    -- 游戏启动时
    gamejolt.session_open()

    -- 游戏运行中定时发送心跳
    gamejolt.session_ping("active")

    -- 游戏退出时
    gamejolt.session_close()

数据的上传与读取（Data）

玩家的所有数据都会被上传到云端（Gamejolt）。
使用gamejolt.data_store(key, data, user_data)存储玩家的数据。

  • key - 你想要存的数据的名字。
  • data - 你想要给这个名字一个什么值。
  • user_data - 填写true即可，表明自动读取玩家的状态。
                
-- 存储玩家数据，令data名为血量，给予20的值。
gamejolt.data_store("HP", 20, true)

使用gamejolt.data_fetch(key, user_data)读取玩家的云端存档。

  • key - 要读取的变量名。
  • user_data - 填写true即可，表明自动读取玩家的状态。
                
-- 读取玩家的HP数据。
local hp = gamejolt.data_fetch("HP", true)
print(hp) -- 20

使用gamejolt.data_fetch_all(user_data)获取玩家所有的云端变量（返回一个表）。

  • user_data - 填写true即可，表明自动读取玩家的状态。
                
-- 读取玩家的所有云端数据。
local details = gamejolt.data_fetch_all(true)

使用gamejolt.data_delete(key, user_data)删除玩家的云端储存变量。

  • key - 要删除的变量名。
  • user_data - 填写true即可，表明自动读取玩家的状态。

使用gamejolt.data_get_keys(user_data)获取玩家现有的所有变量名。

  • user_data - 填写true即可，表明自动读取玩家的状态。

分数排行榜（Scores）

使用gamejolt.fetch_scores_local(table_id)获取玩家本地的所有分数。

  • table_id - 要获取的分数的表。

使用gamejolt.fetch_scores_global(table_id)获取该游戏的所有分数。

  • table_id - 要获取的分数的表。

使用gamejolt.submit_score(score, table_id, sort, extra_data)上传玩家的分数到云端。

  • score - 玩家的分数（字符串类型）。
  • table_id - 要上传到哪个排行榜。
  • sort - 以什么分数进行分类（数字类型）。
  • extra_data - 额外数据，如果需要存特殊的变量类型，可以用这个。
                
-- 上传玩家打出的500分到排行榜TABLE中。
gamejolt.submit_score("500", TABLE, 500)

签名的生成

Gamejolt要求严格的签名。
否则所有网络请求都会被拒绝掉。

签名的要求： 

                
local function generate_signature(request_path)
    local input = gamejolt.base_url .. request_path .. gamejolt.private_key
    return md5.sumhexa(input)
end
Gamejolt的基础url + 专题的url + 秘钥，最后用md5转化。
