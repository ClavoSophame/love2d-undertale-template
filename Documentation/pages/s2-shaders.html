着色器的创建

在LOVE2D中，编写着色器（Shader）通常需要以下几个必须要素：

  • main函数：
    vec4 effect(vec4 color, Image texture, vec2 texture_coords, vec2 screen_coords)
    这是LOVE2D要求的主函数，必须存在。它负责处理每个像素的颜色输出。
  • 返回值：
    effect函数必须返回一个vec4类型的颜色值，通常为texture2D(texture, texture_coords) * color。
  • uniform变量：
    可以声明uniform变量（如extern float time;），用于在Lua中传递参数给Shader。
  • GLSL版本：
    LOVE2D会自动处理GLSL版本声明，通常不需要手动添加#version。 

extern float time;

vec4 effect(vec4 color, Image texture, vec2 texture_coords, vec2 screen_coords)
{
    vec4 pixel = Texel(texture, texture_coords);
    // 这里可以添加自定义特效
    return pixel * color;
}

转换一个着色器

这里我们举例Shadertoy的新建默认Shader 

                
void mainImage( out vec4 fragColor, in vec2 fragCoord )
{
    // Normalized pixel coordinates (from 0 to 1)
    vec2 uv = fragCoord/iResolution.xy;

    // Time varying pixel color
    vec3 col = 0.5 + 0.5*cos(iTime+uv.xyx+vec3(0,2,4));

    // Output to screen
    fragColor = vec4(col,1.0);
}
这种Shadertoy种直接复制粘贴过来的代码是无法直接使用在LOVE2D代码中的。
我们需要使用ShaderConverter.lua库对其进行转换，使其变成正常使用的版本。 
                
local converter = require("Scripts.Libraries.ShaderConverter")

-- 进行shader的转换。
converter.ConvertShader([[

void mainImage( out vec4 fragColor, in vec2 fragCoord )
{
    // Normalized pixel coordinates (from 0 to 1)
    vec2 uv = fragCoord/iResolution.xy;

    // Time varying pixel color
    vec3 col = 0.5 + 0.5*cos(iTime+uv.xyx+vec3(0,2,4));

    // Output to screen
    fragColor = vec4(col,1.0);
}

]])

-- 打印结果，可以查看转换后的shader是否有错误。
converter.printResult()

-- 使用shader。
local shader = love.graphics.newShader(converter.result)
shader:send("iTime", 0)

转换过程的问题

好消息是所有Shader都可以进行转换。
但是坏消息是shader的转换是有很大可能报错的。
所以这就要求了开发者需要利用自己的Shader基础和对库的理解直接对要转化的Shader进行一定的修改。

以下Shader是无法正常转换的： 

                
void mainImage( out vec4 u, in vec2 v )
{
    // Normalized pixel coordinates (from 0 to 1)
    vec2 uv = v/iResolution.xy;

    // Time varying pixel color
    vec3 col = 0.5 + 0.5*cos(iTime+uv.xyx+vec3(0,2,4));

    // Output to screen
    u = vec4(col,1.0);
}
其原因就是写法不标准，直接把fragColor写成了u；把fragCoord写成了v
导致字符串检测失败，无法替换u和v而报错。

这种情况需要优先把u换成fragColor；把v换成fragCoord，这样转换成功的概率为80%+。
最后需要加一个return fragColor的代码。
