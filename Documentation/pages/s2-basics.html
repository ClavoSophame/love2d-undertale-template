学习Lua

SoulEngine使用Lua作为脚本语言，因此需要学习Lua的基础知识。
学习Lua的最好的地方莫过于Lua官方文档。

不过如果你觉得从头系统的学习Lua还是太过于繁琐，更何况还有很多学到的东西是用不到制作游戏中的。
可以直接查看本文档附带的一份Lua教程，里面包含了很多基础的、非常常用的内容。

总而言之，在开始之前，学会Lua是十分重要的

面向过程与面向对象的编程

在SoulEngine中
UI主要使用面向过程的编程。
因为UI如何操作是一个非常简单而确定的逻辑，所以使用面向过程的编程。

玩家、GB、骨头等拥有面向对象的操作。
对于一个GB来说，GB拥有动画、开火逻辑等操作，每一部分都是独立的，因此使用面向对象的编程。

POP（面向过程）的优势：在简单、确定性问题中，“直给”的代码比“设计”的架构更有效。
OOP（面向对象）的优势：在不确定的问题之中，“用合理的架构减少繁琐的更改”。
你需要根据自己项目需求选择适合的编程方式。

对比维度 面向过程 (POP) 面向对象 (OOP)
核心单元 函数 类与对象
代码组织 按步骤分解 按实体和关系抽象
复用性 低（依赖函数库） 高（继承/组合）
扩展性 差（需修改流程） 好（通过多态/接口）
性能 更高（无抽象开销） 较低（对象管理开销）
适合规模 小型、线性问题 大型、复杂系统

面向对象的示例

            
                -- 面向对象（OOP）的示例
                local Bone = {}
                Bone.__index = Bone

                function Bone:init()
                    self.color = {1, 0, 0}
                    self.length = 50
                    self.angle = 0

                    return self
                end

                function Bone:getLength()
                    return self.length
                end
在上述情况下，你创建的每一根骨头都会有上面所定义的init和getLength等方法。
如下，bone1和bone2都拥有自己的方法： 
            
                local bone = Bone:init()
                print(bone:getLength()) -- 输出：50

                local bone2 = Bone:init()
                print(bone2:getLength()) -- 输出：50

面向过程的示例

            
                -- 面向过程（POP）的示例
                function initBone()
                    local bone = {}
                    bone.color = {1, 0, 0}
                    bone.length = 50
                    bone.angle = 0

                    return bone
                end

                function getBoneLength(bone)
                    return bone.length
                end
在上述情况下，你创建的每一根骨头都由你自己决定每一步的过程
如下，bone1和bone2都需要第三方方法来得到相同的结果： 
            
                local bone = initBone()
                print(getBoneLength(bone)) -- 输出：50

                local bone2 = initBone()
                print(getBoneLength(bone2)) -- 输出：50

二者的明显区别在此暂时无法体现出来，只是能方便读者弄清使用方法。

使用骨头的案例，就是为了说明，骨头们也可以拥有独自的长度、角度、以及颜色。
但是，所有的骨头，都有一个共同点，那就是它们都可以对玩家造成伤害、都拥有三个贴图进行首尾相连……
因此，面向对象在此处相当简单。每个骨头只需要执行各自早已写好的逻辑。
