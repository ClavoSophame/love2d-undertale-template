JSON

用于轻量级的数据交换和存储。

DKJSON库的使用

使用代码json.encode将表类型以json格式储存起来。 

                
local json = require("dkjson")  -- 导入 DKJSON 库

local data = {
    name = "Lua",
    version = 5.4,
    features = {"lightweight", "fast"}
}

-- 将 Lua 表编码为 JSON 字符串
local jsonString = json.encode(data)
print(jsonString)  -- 输出：{"name":"Lua","version":5.4,"features":["lightweight","fast"]}
关于state参数，如果设置为{indent = true}那么会变成标准的缩进格式。

使用代码json.decode(str, pos, nullval, ...)读取json的内容，并返回一个表。 

                
local json = require("dkjson")
local jsonStr = '{"name":"Lua","version":5.4,"active":true,"tags":["fast","light"],"license":null}'

-- 解析为 Lua 表
local data = json.decode(jsonStr)

print(data.name)      -- 输出: "Lua"
print(data.tags[1])   -- 输出: "fast"
print(data.license)   -- 输出: nil（因为 null 默认转为 nil）

使用代码json.use_lpeg()函数进行更高效的解析操作。 

                
local json = require("dkjson")

-- 尝试启用 LPEG 解析器（如果 LPEG 库存在）
json.use_lpeg()

-- 后续的 encode/decode 会自动使用 LPEG（如果启用成功）
local data = json.decode('{"key":"value"}')
注：对小规模 JSON 数据，LPEG 的性能优势可能不明显。对嵌入式系统，LPEG 可能增加内存占用（需权衡速度和资源）。
解析几 MB 以上的 JSON 文件时，LPEG 能显著减少耗时。
